Система отчетов в GameClass

В программе есть два типа отчетов, это отчеты оператора (текущий и по последним сеансам) и отчеты менеджера. Отчеты менеджера не видны оператором, все отчеты находятся в главном окне в закладке "Отчеты". Для просмотра нужного отчеты достаточно на него кликнуть мышкой.

С помощью редактора отчетов можно изменить существующие отчеты менеджера или создать свой собственный, не дожидаясь реакции от разработчиков программы. Также можно добавить созданный кем-либо отчет. На сайте программы будут помещаться самые интересные отчеты, созданные пользователями программы GameClass. При просмотре отчета можно использовать фильтр и применять следующие конструкции в полях для фильтрации:
"›2 and ‹100"
"192.168.0.4"
"Internet"
"‹›0"

Как создать нужный отчет самостоятельно?
Для этого нужно знать основы языка SQL, а также изучить формат базы данных GameClass. Просто заходите в редактор отчетов (при этом в GameClass вы должны быть под менеджером) и создаете отчет. При создании отчетов нужно знать, что можно использовать только конструкции типа SELECT, конструкции INSERT, UPDATE,DELETE и EXEC запрещены к использованию и выполняться не будут. В SQL-коде можно использовать шаблоны даты (если в запросе используется дата). Можно использовать следующие шаблоны:
%CURRENTTIME% - текущий момент времени
%CURMONTH-BEGIN% - начало текущего месяца
%TODAY-BEGIN% - начало текущего дня
%SHIFT-BEGIN% - начало текущей смены
%SHIFT2-BEGIN% - начале предыдущей смены
%SHIFT2-END% - конец предыдущей смены
%YESTERDAY-BEGIN% - начало вчерашнего дня
%YESTERDAY-END% - конец вчерашнего дня
%TIME-START% - время установленное вручную в блоке фильтра
%TIME-STOP% - время установленное вручную в блоке фильтра

Как импортировать/экспортировать отчеты?
Используйте кнопки "Импорт" и "Экспорт" редактора отчетов. Экспортируемый отчет записывается в файл с расширением XML. Этот файл можно передать другому пользователю GameClass, и он сможет добавить созданный вами отчет в свои отчеты, проимпортировав этот файл. Внимание! В результате экспорта отчета в XML-файл, в этот файл попадают НЕ РЕЗУЛЬТАТЫ а КОД отчета. У отчетов имеются номера версий. При изменении (улучшении) отчета номер его версии увеличивается. Если отчет импортируется поверх существующего, то существующий заменяется, если номер версии нового больше либо равен версии существующего отчета.

Формат базы данных GameClass и описание таблиц и полей смотрите в разделе "Формат базы данных GameClass"
